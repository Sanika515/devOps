Demonstration: Resolving Merge Conflicts + Documentation
🧠 Scenario (Simple & Real)

Two branches modify the same line → conflict guaranteed.

main → original code
feature-a → change 1
feature-b → change 2
🔹 Step 1: Setup Repository
git clone https://github.com/your-username/demo-repo.git
cd demo-repo

Create file:

vim index.html

Add:

<h1>Welcome to My Website</h1>
git add .
git commit -m "Initial commit"
git push origin main
🔹 Step 2: Create Branch feature-a
git checkout -b feature-a

Edit file:

vim index.html

Change line to:

<h1>Welcome from Feature A</h1>
git add .
git commit -m "Updated heading in feature-a"
git push origin feature-a
🔹 Step 3: Create Branch feature-b (from main)
git checkout main
git checkout -b feature-b

Edit same line:

vim index.html

Change to:

<h1>Welcome from Feature B</h1>
git add .
git commit -m "Updated heading in feature-b"
git push origin feature-b
🔥 Step 4: Merge feature-a into main (No Issue)
git checkout main
git merge feature-a
git push origin main
💥 Step 5: Merge feature-b → CONFLICT
git merge feature-b

👉 Git will throw:

CONFLICT (content): Merge conflict in index.html
🔹 Step 6: Open File to See Conflict
vim index.html

You’ll see:

<<<<<<< HEAD
<h1>Welcome from Feature A</h1>
=======
<h1>Welcome from Feature B</h1>
>>>>>>> feature-b
🔹 Step 7: Resolve Conflict (Manual Fix)

Edit to final version:

<h1>Welcome from Feature A & B</h1>

👉 Remove all <<<<, ====, >>>>

Save → :wq

🔹 Step 8: Mark as Resolved
git add index.html
git commit -m "Resolved merge conflict between feature-a and feature-b"
🔹 Step 9: Push Final Code
git push origin main
📝 Step 10: Document on GitHub (VERY IMPORTANT)

Go to your repo → either:

Option 1: Pull Request Description

If conflict happened via PR:

Write:

Merge Conflict Resolution:
Conflict occurred in index.html due to changes in the same line from both branches.
Resolved by combining both updates into a single meaningful line.

Option 2: README.md Update

Add section:

## Merge Conflict Resolution

- Conflict occurred in index.html
- Reason: Same line modified in two branches
- Resolution: Combined both changes manually

Commit:

git add README.md
git commit -m "Documented merge conflict resolution"
git push origin main
⚡ Final Viva Answer (Say this confidently)

“I created two branches modifying the same line to intentionally generate a merge conflict. Git highlighted the conflict using markers, which I resolved manually by editing the file. I then committed the resolved version and documented the resolution process in GitHub.”
