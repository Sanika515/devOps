# 🧪 **Resolving Merge Conflicts and Documenting the Process in GitHub**

---

# 🎯 Aim

To create a merge conflict, resolve it manually, and document the resolution process using Git and GitHub.

---

# 📘 Core Concept

```text
Same file + Same line + Parallel branches = Merge Conflict
```

---

# ⚙️ Procedure (Step-by-Step)

---

## 🔹 Step 1: Create New Repository Folder

```cmd
cd Desktop
mkdir conflict-lab
cd conflict-lab
git init
```

---

## 🔹 Step 2: Create Initial File

```cmd
echo Original line > file.txt
git add file.txt
git commit -m "Initial commit"
```

---

## 🔹 Step 3: Create Branch feature1

```cmd
git checkout -b feature1
```

---

## 🔹 Step 4: Modify File (feature1)

```cmd
echo Feature1 change > file.txt
git add file.txt
git commit -m "Feature1 modification"
```

---

## 🔹 Step 5: Go to main branch

```cmd
git checkout main
```

👉 If error occurs:

```cmd
git branch -M main
git checkout main
```

---

## 🔹 Step 6: Create Branch feature2 (from main)

```cmd
git checkout -b feature2
```

---

## 🔹 Step 7: Modify SAME line differently

```cmd
echo Feature2 change > file.txt
git add file.txt
git commit -m "Feature2 modification"
```

---

# ⚔️ Conflict Creation

---

## 🔹 Step 8: Merge feature1

```cmd
git checkout main
git merge feature1
```

---

## 🔹 Step 9: Merge feature2 → 💥 Conflict

```cmd
git merge feature2
```

### ✔️ Expected Output

```text
CONFLICT (content): Merge conflict in file.txt
Automatic merge failed
```

---

# ✏️ Conflict Resolution

---

## 🔹 Step 10: Open File

```cmd
notepad file.txt
```

---

## 🔹 Step 11: Conflict View

```text
<<<<<<< HEAD
Feature1 change
=======
Feature2 change
>>>>>>> feature2
```

---

## 🔹 Step 12: Edit Final Version

```text
Feature1 change
Feature2 change
```

👉 Remove all conflict markers

---

## 🔹 Step 13: Save and Commit

```cmd
git add file.txt
git commit -m "Resolved merge conflict between feature1 and feature2"
```

---

# 🔹 Step 14: Push to GitHub

```cmd
git remote add origin https://github.com/your-username/conflict-lab.git
git branch -M main
git push -u origin main
```

---

# 📊 Final Output

## ✔️ File Content

```cmd
type file.txt
```

```text
Feature1 change
Feature2 change
```

---

## ✔️ Git Status

```cmd
git status
```

```text
nothing to commit, working tree clean
```

---

## ✔️ Git Log

```cmd
git log --oneline
```

Shows merge + resolution commit

---

# 📝 Documentation (Write in Practical)

### 🔹 Conflict Description

A merge conflict occurred because both branches modified the same line in `file.txt`.

---

### 🔹 Resolution Steps

1. Conflict identified during merge
2. Opened file and analyzed conflict markers
3. Removed markers (`<<<<<<<`, `=======`, `>>>>>>>`)
4. Combined both changes
5. Saved file
6. Added and committed changes

---

### 🔹 Final Commit Message

```text
Resolved merge conflict between feature1 and feature2
```

---

# 🧠 Key Concepts

* Conflict occurs due to overlapping changes
* Git cannot auto-decide → user resolves manually
* Resolution must be committed

---

Merge conflicts are resolved manually by editing files and committing the final version, ensuring correct integration of parallel changes.
